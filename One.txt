- Be UI-only (no business or ML logic in the frontend)
- Communicate with an existing Python backend using REST APIs
- Replicate the same user flows, pages, and visualizations present in Streamlit

Assume the backend already exposes the required APIs.
Focus on clean component design, API integration, loading/error states, and a production-ready UI structure.
