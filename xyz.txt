I am building an AutoML pipeline with hypothesis-driven feature selection followed by optional feature engineering.

I encountered a prediction-time failure caused by feature engineering and feature selection interacting incorrectly.

Here is the core issue:

- During training, hypothesis testing removed some raw features (e.g., x1, x2, x4) because they were statistically insignificant.
- Feature engineering was then applied and created new features that *depend on those removed raw features* (e.g., x1_x_x2 depends on x1 and x2).
- The preprocessing pipeline was fitted successfully on the engineered feature set.
- At inference time, the pipeline fails because the engineered features cannot be derived—their parent raw features were dropped earlier.

This results in errors like:
"ValueError: columns are missing: {'x1_x_x2', 'x2_x_x5', ...}"

Key constraints:
- Users must provide only raw input features (engineered features are internal).
- Feature engineering must happen inside the pipeline, not in the UI.
- The system must remain production-grade and consistent between training and inference.

Please:
1. Explain why removing raw features before feature engineering causes this failure.
2. Propose a correct pipeline ordering that preserves feature dependencies.
3. Suggest design patterns or guardrails to prevent this issue in AutoML systems.
4. Optionally suggest a pragmatic short-term fix, but clearly distinguish it from the clean long-term design.

Assume scikit-learn pipelines and ColumnTransformer are being used.
