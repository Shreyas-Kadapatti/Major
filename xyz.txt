────────────────────────────────────────────

- Automatically detect target‑correlated features (numeric, categorical, text)

- Detect future/post‑outcome or target‑derived features
- Include simple name‑based heuristics (e.g., columns containing words like target, label, result, outcome)
- Work without training heavy models

Output:
- List of suspected leakage features
- Either warn the user or block training based on severity

Constraints:
- Must be fast
- Model‑agnostic
- Must run BEFORE AutoML training starts

────────────────────────────────────────────

- Feature names must be traceable back to original raw columns
- Raw data must never be overwritten

Goal:
- Every transformed feature must be explainable in terms of its raw data origin
- The preprocessing pipeline must be reproducible and auditable

- Compute SHAP only once per trained model

Goal:
- Low‑latency explainability even for large datasets and complex models
